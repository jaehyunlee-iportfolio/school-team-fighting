# 출장신청서 자동 생성기 — 기획문서

> **프로젝트명**: school-team-fighting  
> **한줄 요약**: 건국대 사업단 D-4 출장비 CSV + 결재 서명 이미지 → A4 출장신청서 PDF 자동 생성  
> **작성일**: 2026-04-24  
> **기술 스택**: Next.js 15 (Turbopack), TypeScript, Tailwind CSS, shadcn, react-pdf, papaparse, JSZip  
> **배포 타겟**: Vercel (정적 빌드, 서버 사이드 없음 — 모든 처리는 브라우저에서 수행)

---

## 1. 프로젝트 배경

### 1.1 해결하려는 문제

건국대 사업단에서 **`★2025_세부비목별집행내역서(과세)` 엑셀의 "D-4. 출장비" 시트**에 기록된 출장 건마다 **출장신청서 PDF**를 수기로 작성해야 하는 반복 업무가 있다.

- 해당 시트에는 약 **100건 이상**의 출장 행이 존재 (2025.05 ~ 2026.01 범위)
- 행 1건 = 출장신청서 1장
- 수기 작성 시: 거래처(이름) 복사, 일자 복사, 출장지·목적 복사, 서명 이미지 삽입 등 → **건당 3~5분** 소요

### 1.2 목표

> **CSV 파일 + 서명 이미지 2장만 올리면, 전 행에 대한 출장신청서 PDF를 한 번에 자동 생성·다운로드할 수 있는 웹 도구**

## 2. 입력 데이터 분석

### 2.1 CSV 원본 (D-4 출장비 시트)

- **파일명 예시**: `★2025_세부비목별집행내역서(과세)_건국대사업단_(주)아이포트폴리오 - D-4.출장비.csv`
- **헤더 구조**: 2행 병합 헤더 (상행 + 하행)
  - 상행 예: `사용일자 | 비고(증빙번호) | 집행일자 | 거래처 | 지출금액 | … | 출장지 | 지급방법 | 집행 기관명 | …`
  - 하행은 `지출금액` 아래 세부열(`공급가액`, `부가세`, `합계금액`)과 담당자명 등
- **데이터 행**: `D-4-3-1` ~ `D-4-3-107` (약 95건, 소계/정의/주의사항 행 제외)
- **주요 열 → PDF 필드 매핑**:

| CSV 열 | PDF 필드 | 비고 |
|--------|----------|------|
| `거래처` | 작성자 성명 / 출장 인원 | 첫 번째 사람(`,` `/` 구분자 기준) = 기안자(성명). 전원 = 출장 인원 |
| `사용일자` | 출장 기간 | `"2025. 7. 1 ~ 2025. 7. 31"` 범위 형식. 날짜 1개면 동일 날짜로 시작~끝 자동 보정 + 경고 |
| `집행 기관명` | 작성자 소속 | `(주)아이포트폴리오` 또는 `(사)디지털미디어교육콘텐츠 교사연구협회` 등 |
| `출장지` | 출 장 지 | 장소명(학교, 지역, 행사명) |
| `사용내역(수령인)` | 출장 목적 | `1. 출장자명(…) 2. 출장비 산출내역: … 3. 출장내용…` 형태의 자유텍스트 |

> **참고**: `집행일자`와 `합계금액` 열은 사용하지 않음 (무시)

### 2.2 서명 이미지

프로젝트에 **4개의 서명 이미지**가 보관되어 있으며, 집행기관에 따라 결재 구성이 달라진다:

| 집행기관 | 결재자1 (직위) | 결재자2 (직위) | 서명 파일 |
|----------|---------------|---------------|----------|
| **(주)아이포트폴리오** (iPF) | 팀장 | 본부장 | `ipf/서명이미지_결재자1_아포폴_팀장.png`<br>`ipf/서명이미지_결재자2_아포폴_본부장.png` |
| **디미교연** | 사무국장 | 대표이사 | `디미교연/서명이미지_결재자1_디미교연_사무국장.png`<br>`디미교연/서명이미지_결재자2_디미교연_대표이사.png` |

- **기안자(맨 왼쪽 칸)**: 서명 이미지가 아닌, 거래처 열의 **맨 앞 사람 이름(최대 3자)**을 **손글씨 폰트(나눔펜스크립트)**로 표시
- **결재자1·2**: 사용자가 업로드한 PNG/JPG 서명 이미지를 삽입

### 2.3 PDF 양식 (출력물)

Mock PDF 기준 A4 단면, 구성:

```
┌───────────────────────────────────────────────┐
│  출장신청서                    결 │ 기안자 │ {결재1} │ {결재2} │
│                              재 │  서명  │  서명   │  서명   │
│                                                                │
│  작성자 소속  │  {집행기관명}                                   │
│  작성자 성명  │  {거래처 → 첫 번째 사람}                        │
│                                                                │
│  아래와 같이 출장을 신청합니다.                                  │
│                                                                │
│  출장 인원    │  {거래처 → 전원}                                │
│  출장 기간    │  {사용일자 범위 "YYYY. M. D ~ YYYY. M. D"}       │
│  출 장 지     │  {출장지 열}                                    │
│  출장 목적    │  {사용내역(수령인) — 높은 셀}                    │
│                                                                │
│                      {소속 (중앙, 굵게)}                        │
└───────────────────────────────────────────────┘
```

## 3. 시스템 아키텍처

### 3.1 전체 흐름

```
[사용자]
   │
   ├─ 1. CSV 파일 선택 (.csv)
   ├─ 2. 결재 서명 이미지 2장 (PNG/JPG, 선택)
   ├─ 3. 결재란 오버라이드 (auto / iPF / 디미교연)
   │
   ▼
[브라우저 — parseD4.ts]
   │
   ├─ papaparse로 CSV 파싱
   ├─ 2행 병합 헤더 해석 (buildMergedHeaderKeys)
   ├─ 데이터 행 필터링 (isDataRow)
   ├─ 행별 TripRow 구조체 생성
   │   ├─ 거래처 → 첫 토큰 = 작성자명 (parseName.ts)
   │   ├─ 집행기관명 → iPF/디미교연/unknown 분류 (labels.ts)
   │   ├─ 사용내역 → 출장 목적 텍스트
   │   └─ 빈 필드 → fieldWarnings
   │
   ▼
[브라우저 — trip-tool.tsx]
   │
   ├─ Step 1: 자료 (파일 + 모드 선택)
   ├─ Step 2: 검토 (테이블 + 미리보기)
   └─ Step 3: 끝 (다운로드 완료)
         │
         ├─ 바로 생성 모드: react-pdf → Blob → JSZip → .zip 다운로드
         └─ 미리보기 모드: 행별 PDF 미리보기 iframe → 개별 다운로드
```

### 3.2 핵심 모듈

| 파일 | 역할 |
|------|------|
| `src/lib/csv/parseD4.ts` | CSV 파싱, 2행 헤더 병합, `TripRow[]` 생성 |
| `src/lib/names/parseName.ts` | 거래처/사용내역에서 이름 추출, 기안 서명용 최대 3자 생성 |
| `src/lib/approval/labels.ts` | 집행기관명 → iPF/디미교연 분류, 결재 직위 라벨 반환 |
| `src/lib/pdf/register-pdf-fonts.ts` | Pretendard + NanumPen(손글씨) 폰트 등록 (브라우저, 1회) |
| `src/components/pdf/business-trip-document.tsx` | react-pdf `<Document>` 컴포넌트. A4 양식 레이아웃 |
| `src/components/trip-tool.tsx` | 전체 플로우 UI (3단계 위저드) |

### 3.3 폰트

| 폰트 | 용도 | 파일 |
|------|------|------|
| Pretendard Regular | 본문 + UI | `public/fonts/Pretendard-Regular.otf` |
| NanumPen Script | 기안자 서명(손글씨 느낌) | `public/fonts/nanum-pen-korean.woff2` |

## 4. UI 플로우 (3단계 위저드)

### Step 1 — 자료

| 요소 | 설명 |
|------|------|
| **생성 모드** | `바로 생성(추천)` / `미리보기` — 좌우 2열 카드 |
| **CSV 파일** | D-4 출장비 시트를 CSV(UTF-8)로 저장한 파일 |
| **결재 서명 1·2** | 이미지(PNG/JPG). 옵션 — 없으면 `(서명)` 텍스트 대체 |
| **결재란 머리** | auto(집행기관명으로 자동) / iPF 강제 / 디미교연 강제 |
| **다음** | CSV 파싱 후 Step 2로 |

### Step 2 — 검토

| 요소 | 설명 |
|------|------|
| **행 요약 테이블** | 모바일=카드 리스트, 데스크톱=테이블(#, 이름, 기안3자, 집행기관, 결재제목, 상태) |
| **경고 배지** | 빈칸/누락 의심 행에 `빈칸` 뱃지, 양호 행에 `양호` 뱃지 |
| **미리보기 모드 한정** | PDF iframe + 행 번호 선택 UI |
| **하단 액션** | `이전(자료)` / `ZIP으로 받기` 또는 `전부 PDF로(행마다)` |

### Step 3 — 끝

| 요소 | 설명 |
|------|------|
| **완료 메시지** | 생성 건수, 다운로드 안내 |
| **처음으로** | 전체 초기화, Step 1로 |

## 5. PDF 생성 로직

### 5.1 행 → PDF 매핑

```
CSV 행 (TripRow)
  ├─ writerName          → 작성자 성명, 출장 인원
  ├─ drafter3            → 기안 서명란 (NanumPen 폰트, 최대 3글자)
  ├─ orgName             → 작성자 소속, 페이지 하단 조직명
  ├─ periodText          → 출장 기간 (사용일자 범위, "YYYY. M. D ~ YYYY. M. D")
  ├─ outPlace            → 출장지
  ├─ purposeText         → 출장 목적 (사용내역 전문)
  ├─ approver1 / 2       → 결재란 제목 (팀장/본부장 또는 사무국장/대표이사)
  └─ approver1Src / 2Src → 결재 서명 이미지 (data URL)
```

### 5.2 결재란 구조 (세로 '결재' + 3열 × 2행)

```
┌──────┬──────────┬──────────┬──────────┐
│      │  기안자  │ {결재1}  │ {결재2}  │  ← 직위
│  결  ├──────────┼──────────┼──────────┤
│  재  │  (3자)   │  서명img │  서명img │  ← 서명
└──────┴──────────┴──────────┴──────────┘
```

### 5.3 ZIP 구성 (바로 생성 모드)

```
출장신청서_20260424.zip
  ├─ 출장신청서_001_이영규.pdf
  ├─ 출장신청서_002_채영지.pdf
  ├─ 출장신청서_003_채영지.pdf
  └─ ...
```

```

    │

    │       ├─ firstTokenFromPartnerCell()  (거래처 셀의 첫 토큰)

    ├─ drafterSignatureGraphemes(name, 3)  → drafter3
    ├─ detectApprovalGroup(orgName)         → ipf / dimi / unknown

    └─ fieldWarnings[]                      → 빈 필드 경고

    │
    ▼

    │
    ▼

```

## 7. 이름 추출 규칙 (parseName.ts)

출장비 CSV의 **거래처** 열과 **사용내역(수령인)** 열에서 작성자(기안자) 이름을 추출한다:

1. **우선순위 1 — 거래처 열**: `채영지, 이창환` → 쉼표/슬래시/가운뎃점으로 분리 → 첫 번째 토큰 `채영지`
2. **우선순위 2 — 사용내역**: `1. 출장자명(채영지)` 패턴 매칭 → 괄호 안 이름
3. 양쪽 다 못 찾으면: `nameSource: "none"`, `hasEmpty: true`

**기안 서명**: 추출된 이름의 **앞 3자** → NanumPen 폰트로 PDF에 표시

## 9. 기술 상세

### 9.1 CSV 파싱 특수 처리

- **2행 병합 헤더**: 상행(`사용일자`, `거래처`, `지출금액`, …)과 하행(`공급가액`, `부가세`, …)을 병합하여 고유 키 생성
- **데이터 행 판별**: 날짜 패턴(`YYYY-MM-DD`, `YYYY. M. D`, `YYYY. M. D ~ YYYY. M. D`), `D-4` 포함, 한글 2자 이상 등으로 소계/정의/주의사항 행 제외
- **사용일자 정규화**: 범위(`~` 포함)면 그대로, 단일 날짜면 `"YYYY. M. D ~ YYYY. M. D"` 형태로 자동 보정 + 경고 표시
- **중복 키 처리**: `사용내역(수령인)__2` 식으로 자동 넘버링

### 9.2 PDF 렌더링

- **라이브러리**: `@react-pdf/renderer` v4.5 — 브라우저에서 직접 PDF Blob 생성
- **용지**: A4 (595.28 x 841.89 pt), 20mm 여백
- **결재란**: 세로 `결/재` 셀 + 3열(기안자·결재1·결재2) × 2행(직위·서명)
- **출장 목적 셀**: minHeight 112pt로 긴 텍스트 수용
- **하단 조직명**: 페이지 최하단 중앙

### 9.3 번들링·다운로드

- **바로 생성**: 행별 `pdf(<Document>).toBlob()` → `JSZip`으로 묶기 → `<a download>` 트리거
- **미리보기**: 행 1건 → `URL.createObjectURL(blob)` → iframe.src, 이후 개별 다운로드

## 10. 제약사항 및 전제

| 항목 | 내용 |
|------|------|
| **순수 프런트엔드** | 서버/DB 없음. 모든 파싱·PDF 생성은 브라우저에서 수행 |
| **파일 저장** | Vercel 배포 시 서버 파일 쓰기 불가 → `<a download>` 로 다운로드만 가능 |
| **CSV 인코딩** | UTF-8 권장. EUC-KR일 경우 깨질 수 있음 |
| **PDF 폰트** | Pretendard Regular만 번들 (Bold 미포함). NanumPen은 기안란 전용 |
| **서명 이미지** | 사용자가 매번 업로드해야 함 (보안상 로컬 저장 불가) |
| **행 수 상한** | 100건 이상도 가능하나, 브라우저 메모리에 의존 (200건 이상 시 속도 저하 가능) |
